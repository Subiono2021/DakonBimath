📜 1. Jejak Historis & Filosofi Dakon

Dakon (atau Congklak) adalah salah satu permainan papan tertua di dunia yang berakar dari tradisi Mancala.

  • Asal Usul: Jejak arkeologis ditemukan di Yordania (7000 SM) dan menyebar ke Nusantara melalui jalur perdagangan kuno.
  • Filosofi: Simbol sawah & benih → melatih manajemen sumber daya, strategi, dan ketelitian.
  • Simbolisme Astronomi: Gerakan memutar biji melambangkan siklus peredaran benda langit dan kontinuitas hidup.
  • Filosofi & Psikomotorik: Simbol sawah & benih → melatih koordinasi mata-tangan, presisi distribusi biji secara satu-per-satu, serta ketelitian dalam menghitung cepat secara visual.

💡 Tantangan: Kalahkan PC dalam ≤ 2 ronde. Pastikan di akhir permainan lumbung PC berisi nol biji (kosong) sementara Anda menguasai mayoritas biji. 🏆 HADIAH SPESIAL jika berhasil!

💡 Insight BiMath: Dakon adalah bukti bahwa nenek moyang kita telah mempraktikkan Aritmetika Modulo jauh sebelum istilah tersebut diformalkan dalam matematika modern.

2. 🧮 Representasi Matematis Papan

Papan Dakon dimodelkan sebagai vektor 14-dimensi dalam ruang state $\mathbb{Z}_{\geq 0}^{14}$:

board = [a₀, a₁, a₂, a₃, a₄, a₅, s₁, b₇, b₈, b₉, b₁₀, b₁₁, b₁₂, s₂]
  • 0-5 Lubang kecil Pemain (Anda)
  • 6 Lumbung (Store) Anda
  • 7-12 Lubang kecil CPU
  • 13 Lumbung (Store) CPU
📊 Invarian Permainan:
$\sum_{i=0}^{13} \text{board}[i] = 72$ (Total biji tetap konstan).

3. 🕸️ Topologi Graf & Aritmetika Modulo

Aliran biji mengikuti struktur Graf Siklus Terarah (Directed Cycle Graph) $C_{14}$.

Next_Index = (Current_Index + 1) mod 14

Secara matematis, pergerakan ini adalah translasi dalam grup siklik $\mathbb{Z}_{14}$, namun dengan batasan (skip condition):

  • Pemain $P_1$ melewati indeks 13 ($x \neq 13$).
  • CPU $P_2$ melewati indeks 6 ($x \neq 6$).

4. 🔄 Dinamika Transisi State

Mekanisme Penaburan (Sowing)

Fungsi transisi $f(S, i)$ memetakan state $S$ ke $S'$ berdasarkan lubang asal $i$.

Aturan "Nembak" (Capture)

Terjadi jika biji terakhir jatuh di lubang kosong sisi pemain ($0 \leq i \leq 5$) dan lubang lawan ($12-i$) tidak kosong:

s₁ ← s₁ + 1 + board[12-i]
board[i] ← 0, board[12-i] ← 0

5. 🎯 Strategi AI: Bounded Minimax

CPU mencari langkah optimal menggunakan pohon keputusan dengan kedalaman $d=3$.

E(board) = board[13] - board[6]

Fungsi evaluasi ini mengarahkan CPU untuk memaksimalkan skor relatif. Dengan 6 pilihan langkah per turn, AI mengevaluasi hingga $6^3 = 216$ kemungkinan state di masa depan.

6. 🔄 Model Partisi Ronde

Transisi antar ronde menggunakan algoritma pembagian bilangan bulat (Integer Division):

Jumlah_Lubang_Aktif = floor(Total_Biji / 6)
🔥 Lubang Mati: Jika sisa biji < 6, lubang tersebut dinyatakan "terbakar", mengurangi dimensi ruang state pada iterasi berikutnya.

Berdasarkan analisis mendalam terhadap kelemahan AI dan struktur matematis permainan, berikut adalah kunci kemenangan absolut (lumbung CPU = 0, ronde 1 atau 2):

💡 Eksploitasi Kelemahan Minimax CPU

  • CPU hanya melihat 3 langkah ke depan. Ia TIDAK bisa merencanakan efek transisi ronde yang terjadi setelah giliran ke-3.
  • Fungsi evaluasi $E = s_2 - s_1$ membuat CPU rakus mengejar selisih lumbung sesaat. Ia tidak sadar bahwa mengisi lumbung Anda sedikit bisa berakibat fatal di ronde berikutnya.
  • Strategi: Bangun posisi yang "buruk" menurut CPU (selisih lumbung kecil) tetapi dalam 2 langkah lagi akan mengosongkan semua biji lawan melalui capture berantai.

🎯 Target Utama: $s_2 = 0$ di Akhir Ronde 1 atau 2

Permainan memberikan hadiah spesial jika board[13] == 0 && round <= 2. Artinya Anda harus membuat lumbung CPU benar-benar kosong sebelum ronde ke-3 bergulir. Caranya:

  1. Paksa Dead Pits di Sisi CPU: Di akhir ronde, sisa biji di lumbung CPU ($s_2$) dibagi 6. Jika $s_2 < 6$, maka di ronde berikutnya tidak ada lubang CPU yang aktif (semua jadi 🔥). Ini adalah kondisi "mati total" bagi CPU.
  2. Gunakan Capture (Nembak) secara Agresif: Setiap kali Anda berhasil menembak, Anda memindahkan biji lawan langsung ke lumbung Anda. Rencanakan agar semua biji CPU habis tertembak sebelum ronde berakhir.
  3. Manfaatkan aturan "main lagi" (extra turn): Jika biji terakhir jatuh di lumbung Anda, Anda mendapat giliran tambahan. Ini bisa Anda gunakan untuk terus menyerang tanpa memberi lawan kesempatan.

📐 Rumus Praktis di Lapangan

// Cek apakah langkah dari lubang i akan berakhir di lumbung Anda (extra turn)
willLandInStore(i) = ( (board[i] + i) mod 14 ) == 6
// Cek potensi capture: setelah menabur, apakah lubang tujuan kosong dan lubang lawan (12-i) berisi > 0 ?

Kombinasikan kedua kondisi di atas. Idealnya: pilih lubang yang (a) memberi extra turn dan (b) segera diikuti capture pada giliran kedua (karena Anda main lagi).

⚡ Contoh Skenario Kemenangan 2 Ronde

  • Ronde 1: Anda bermain sangat ofensif, melakukan capture berulang. Di akhir ronde, semua biji CPU (awalnya 36 biji) habis berpindah ke lumbung Anda. $s_2^{(1)} = 0$, $s_1^{(1)} = 72$.
  • Transisi ke Ronde 2: Karena $s_2=0$, CPU tidak memiliki biji untuk mengisi lubang kecilnya. Keenam lubang CPU menjadi dead.
  • Ronde 2: Giliran Anda mulai. Anda tinggal menabur dari lubang Anda sendiri. Lumbung CPU tetap 0. Setelah giliran Anda selesai, permainan berakhir dengan kemenangan mutlak Anda.
